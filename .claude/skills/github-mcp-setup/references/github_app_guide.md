# GitHub App Authentication Guide

## Why GitHub Apps Over Personal Access Tokens?

**GitHub Apps** provide superior security and management compared to Personal Access Tokens (PATs):

- **Granular Permissions**: Apps request only the permissions they need
- **Audit Trail**: GitHub logs all actions performed by the app
- **Revocable Access**: Uninstall the app instantly to revoke all permissions
- **Expiration Management**: Private keys can be rotated without manual token regeneration
- **Per-Installation**: Different installations can have different permissions

## GitHub App Credentials Explained

### App ID
- Numeric identifier for your GitHub App
- Non-sensitive; safe to commit
- Found in app settings under "About"

### Private Key
- PEM-format cryptographic key for signing requests
- **MUST be kept secret** — store in `.github/app-private-key.pem` and add to `.gitignore`
- Automatically generated by GitHub; can be rotated by generating a new key
- Referenced via `GITHUB_APP_PRIVATE_KEY_PATH` to avoid hardcoding

### Installation ID
- Numeric ID for a specific installation of your app on a user/org
- Non-sensitive; identifies where the app is installed
- Each org/user with the app installed has a unique Installation ID
- Found in app installation settings URL

## Permission Matrix

Configure app permissions based on your use case:

| Use Case | Repository | Account |
|----------|-----------|---------|
| **Code Review** | Contents (R), PRs (R/W), Issues (R) | Email (R) |
| **PR Automation** | Contents (R), PRs (R/W), Issues (R/W) | Email (R) |
| **Issue Tracking** | Issues (R/W), Discussions (R/W) | — |
| **Full Automation** | Contents (R/W), PRs (R/W), Issues (R/W) | Email (R) |

*R = Read, R/W = Read/Write*

### Permission Descriptions

**Repository Permissions:**
- **Contents**: Read/write repository code, create branches, manage files
- **Pull Requests**: Read/write PRs, create, comment, manage reviews
- **Issues**: Read/write issues, create, comment, manage labels
- **Discussions**: Read/write repository discussions

**Account Permissions:**
- **Email**: Read user email addresses

## Security Best Practices

1. **Rotate Keys Regularly**: Generate new private keys periodically, then delete old ones
2. **Use Environment Variables**: Never hardcode credentials; use `.env` with path references
3. **Gitignore Sensitive Files**: Always `.gitignore` the private key file
4. **Principle of Least Privilege**: Grant only permissions your workflow needs
5. **Monitor Installations**: Review installed apps regularly in GitHub settings
6. **Restrict Installation Scope**: Install apps only where needed (your repos, not all orgs)

## Revoking Access

To revoke GitHub App access:

1. Go to [Installed GitHub Apps](https://github.com/settings/installations)
2. Click **Configure** next to the app
3. Click **Uninstall** at the bottom
4. Access is immediately revoked

To rotate credentials:

1. Go to app settings → **Private keys**
2. Click **Generate private key** to create a new one
3. Update `.env` with the new key file path (if location changed)
4. Delete the old private key
5. Old key can no longer sign requests

## Troubleshooting Commands

**Verify app credentials are loaded:**
```bash
echo $GITHUB_APP_ID
echo $GITHUB_APP_PRIVATE_KEY_PATH
echo $GITHUB_INSTALLATION_ID
```

**Check private key file exists and is readable:**
```bash
ls -l $(cat .env | grep GITHUB_APP_PRIVATE_KEY_PATH | cut -d= -f2)
```

**View MCP configuration:**
```bash
cat .mcp.json
```
